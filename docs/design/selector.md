# 종목 선정 정책 (selector)

`selector.py`의 hot pick 선정은 **2단계**로 동작한다.

1. **1차 — 검색 상위**: 네이버 검색 상위(`finance.naver.com/sise/lastsearch2.naver`) 약 30개 중
   dedup 통과분을 **검색 순위 그대로** `top_n`개 선택.
2. **2차 — 보강**: 1차가 모자라면 부족분만 시총 상위 200 풀에서 z-score 룰로 채움.
3. **0건 허용**: 1·2차가 모두 dedup에 걸리면 **강제로 채우지 않고 0건을 반환**한다.

## 설계 의도

- 목표는 "시총·추세 상위"가 아니라 **사람들이 오늘 실제로 검색·관심 갖는 종목**이다.
  검색 상위는 등락률·테마·뉴스 카탈리스트가 모두 압축된 신호로 본다.
- dedup은 의도적으로 유지한다(다양한 종목 커버리지가 목적). 검색 상위라도 최근 30일 내
  분석된 종목이면 skip. 따라서 **0건은 버그가 아니라 정책의 자연스러운 결과**다.

## 유지보수 주의

- 1차 우선 → 2차 보강의 순서를 깨지 말 것.
- dedup 윈도우(30일, `recently_analyzed_tickers(days=30)`)는 자동으로 줄이지 말 것.
- 0건 결과를 오류로 처리하지 말 것 — 로그 한 줄로 끝내고 다음 정각에 재시도한다.
- 검색 상위 페이지는 ~28종목만 노출된다(`limit 30`이어도 그 이상은 못 가져옴).
  더 필요하면 다른 페이지/API 발굴이 필요하다.
- 검색 상위는 시총 상위와 거의 부분집합 관계라, 시총 풀 밖에서 합류하는 종목은 0~3개 수준.
