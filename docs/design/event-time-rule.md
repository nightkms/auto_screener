# 이벤트 시간/시제 룰

분석 보고서에 들어가는 모든 이벤트는 **시간 가치(time decay) + 시제 + 가격 반영 여부**를
검증해야 한다. 옛 사건을 현재 리스크/카탈리스트로 그대로 끌어오지 않는다.

## 배경

LLM이 공시 제목의 키워드를 **과거의 동일 키워드 사건(학습 지식)** 과 자동으로 연결해,
미확인 공시를 단정적인 리스크로 과대 서술하는 실패 모드가 있었다.
**같은 키워드 ≠ 같은 사건** — 공시 본문(rcept_no) 미확인 시 단정 금지.

## 이벤트 시간 윈도우 표준

| 이벤트 유형 | 권장 윈도우 | 비고 |
|---|---|---|
| 일회성 충격 (영업정지·리콜·횡령) | 12~18개월 | 후속 미확정일 때만 살아있는 리스크 |
| 소송·규제 진행 | 진행 중인 동안만 | 종결 시 이미 반영 |
| 구조적 변화 (M&A·사업철수·대규모 capex) | 24~36개월 | P&L 영향 2~3년 |
| 거버넌스·회계 (감사의견·분식) | 24개월+ | 멀티플 디스카운트 장기 |
| 거시·산업 트렌드 | 36~60개월 | 베이스라인 |
| 분기 실적·가이던스 | 직전 4~6분기 | 그 이상은 베이스라인 |

## 이벤트마다 의무 메타 (분석 출력에 명시)

1. **시점** — 발생일 또는 예정일 (YYYY-MM 또는 DART 공시일 8자리)
2. **현재 상태/확정도** — active/closed/unresolved (과거) 또는 confirmed/expected/rumor (미래)
3. **가격 반영** — priced-in/pending/partial (과거 이벤트만)

세 필드 중 하나라도 비면 본문 인용 금지.

## 키워드 자동 매칭 금지

- 공시 제목에 같은 키워드가 있어도 옛 동일 키워드 사건과 자동 연결 금지.
- 공시 본문(rcept_no) 미확인 시 "본문 미확인" 명시 후 추정 수위 낮출 것.
- 베이스라인 사실(주력 사업의 장기 트렌드)로만 짧게 언급은 OK.

## 프롬프트 적용 위치

- `prompts/risk.txt` — 18개월 윈도우 + 3필드 의무 + 키워드 자동매칭 금지
- `prompts/catalyst.txt` — 동일 룰 + 미래 이벤트는 윈도우 무관(단 확정도 표기 필수)
- `prompts/industry.txt` — 36~60개월 허용하되 "현재까지 유효한 추세" 명시 의무
- `prompts/synthesizer.txt` — 시제 검증 후처리 필터(메타 누락 이벤트는 본문·매트릭스에서 제외)
